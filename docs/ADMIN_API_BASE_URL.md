# التحكم في عنوان الـ API ووضع التطوير من اللوحة (Admin)

التطبيق يقرأ من استجابة الـ bootstrap:
- **api_base_url**: عنوان الـ API ويحفظه محلياً.
- **development_mode**: عند true يظهر بانر "وضع التطوير" وشاشة التطوير في التطبيق.

## استجابة GET /api/config/bootstrap

أضف الحقلين في الـ JSON (مع theme, splash, onboarding, …):

```json
{
  "theme": { ... },
  "splash": { ... },
  "onboarding": [ ... ],
  "api_base_url": "https://api.zayer.com",
  "development_mode": true
}
```

| الحقل             | النوع   | الوصف |
|-------------------|--------|--------|
| `api_base_url`    | string | عنوان أساس الـ API (بدون / في النهاية). مثال: `https://api.zayer.com` أو `http://localhost:8000` |
| `development_mode`| boolean| عند true تظهر شاشة/بانر وضع التطوير في التطبيق |

- إذا وُجد وقيمته غير فارغة، التطبيق يستخدمه لجميع طلبات الـ API ويحفظه للجلسات القادمة.
- إذا غاب أو كان فارغاً، يبقى التطبيق على العنوان الافتراضي (من بيئة البناء أو المحفوظ سابقاً).

## في لوحة التحكم (Laravel)

1. إضافة حقل في جدول/إعدادات التكوين العام (مثلاً في نفس مكان theme أو في صفحة "إعدادات التطبيق").
2. عند إرجاع الـ bootstrap (في الـ Controller أو الـ Resource)، أدرج قيمة هذا الحقل كـ `api_base_url`.
3. المستخدم يضع في اللوحة عنوان الـ API (مثلاً `https://api.zayer.com`) ويحفظ؛ الاستجابة التالية لـ GET /api/config/bootstrap ستحتوي القيمة الجديدة وسيتبناها التطبيق تلقائياً.

## ملاحظات

- **أول تشغيل:** التطبيق يستخدم عنواناً افتراضياً (من dart-define أو localhost على الويب) لطلب الـ bootstrap. بعد استلام `api_base_url` يتبناه ويحفظه.
- **تغيير اللوحة:** عند تغيير القيمة في اللوحة، التطبيق يحدّث عند الطلب التالي للـ bootstrap (مثلاً عند فتح التطبيق أو عمل pull-to-refresh إن كان يعيد جلب الـ config).
